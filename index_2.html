Simplify Your Pages!

  • Reduce the number of widgets in your pages

  • Reduce the size of your pages

  • Limit the size of your lists and tables

  • Consider using pagination

Reduce the number of requests you make

  • Combine and minify scripts

  • Combine and minify CSS

  • Do not include scripts or style sheets inside your pages

  • Consider using the multi-page template

  • Consider placing scripts and style sheets directly in the page

Advantages of the multi-page template

  • Load all pages with a single request

  • Reduces the number of HTTP requests

  • Instant page loads

  • Smoother transitions

But it has its limitations...

Disadvantages of the multi-page template

  • Slower initial page download

  • Can lead to a very large dom

  • Consumes more memory

  • Cannot load multi-page documents via Ajax nav

  • Not good for large numbers of pages

  • Organization can suffer from stuffing everything in a single file

Single Page Template

Disadvantages of the single page template

  • Entire document send with every request

  • Only uses first div with a data-role of page

  • Unused information sent with every request

  • Extra information sent leads to extra processing and traversal of the Dom on each request

  • Added HTTP requests

The enhanced option

  • Tells the widget you have provided the markup in a pre-rendered state

  • Avoids all Dom manip during widget creation

  • All thats done on widget create is attaching event handlers

  • Leverage your server to generate markup to speed up page loads

Widgets which currently support enhanced option

  • Textinput

  • Button

  • Checkboxradio

  • Controlgroup

  • Page

  • Collapsible

  • Collapsible Set

  • Popup

  • Table

The defaults flag

  • Checking data-attributes is an expensive operation

  • setting data-defaults=true will by pass all data-attribute option checking

  • Useful when the defaults are what you want

  • Also use it when calling widget directly through Javascript rather then auto-init

$.mobile.window

  • Stored refrence to $( window )
  • Useful to avoid creating a jQuery object on every use

$.mobile.document

  • Stored refrence to $( document )
  • Useful to avoid creating a jQuery object on every use

  • Used internally

  • especially useful for attaching delegated events

  • Don't use inside widgets. Use this.document and this.window

Don't use auto initialization

  • It's and easy shortcut but you sacrifice performance

  • The framework has to traverse the entire Dom looking for widgets to initialize

  • attribute selectors are very slow

  • Call all your widgets from scripts just like you would normally ( ex: using jQuery UI )

Dont use button and listview widgets

  • These widgets are pure decoration no event bindings

  • Just add the classes yourself

  • avoide the javascript all together

  • Takes a little work because you want the css just not the js

Don't use client side mvc in webapps or sites

  • your server is much faster leverage your server for your templating

  • The device is the slowest link in your chain

  • hybrid apps can be an execption when no server is invloved
